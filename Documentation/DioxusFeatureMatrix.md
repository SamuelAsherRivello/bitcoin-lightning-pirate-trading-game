# Dioxus Feature Matrix

Keep this file updated as development continues. When routes, platform support, cache behavior, or Dioxus feature usage changes, update the matching rows in the same change.

| # | Feature | Status | Current template usage | Docs |
| - | ------- | ------ | ---------------------- | ---- |
| 1 | Components | ✅ | Shared UI uses `#[component] fn Name(...) -> Element` in components such as [`Page`](../packages/ui/src/client/components/page.rs), [`PageHeader`](../packages/ui/src/client/components/page_header.rs), [`PageFooter`](../packages/ui/src/client/components/page_footer.rs), and routed pages such as [`Page01`](../packages/ui/src/client/pages/page01.rs). | [Components](https://dioxuslabs.com/learn/0.7/essentials/ui/components/) |
| 2 | RSX markup | ✅ | Pages and components render HTML-like UI through `rsx!`; see [`TemplatePage`](../packages/ui/src/client/pages/template_page.rs). | [RSX](https://dioxuslabs.com/learn/0.7/essentials/ui/rsx/) |
| 3 | Signals | ✅ | [`AppLayout`](../packages/ui/src/client/mod.rs) provides theme, language, template data load request, cached result, and toast signals. | [Signals](https://dioxuslabs.com/learn/0.7/essentials/basics/signals/) |
| 4 | Context state | ✅ | The app uses `use_context_provider` and `use_context::<Signal<T>>()` for shared UI state. | [Global context](https://dioxuslabs.com/learn/0.7/essentials/basics/context/) |
| 5 | Router | ✅ | [`Route`](../packages/ui/src/client/mod.rs) maps `/` to `Page01`, `/page-02` to `Page02`, and `/page-03` to `Page03`. | [Routes](https://dioxuslabs.com/learn/0.7/essentials/router/routes/) |
| 6 | Router links | ✅ | [`PageHeader`](../packages/ui/src/client/components/page_header.rs) uses router-aware `Link` components. | [Navigation](https://dioxuslabs.com/learn/0.7/essentials/router/navigation/) |
| 7 | Asset macro | ✅ | Entry CSS, app-shell CSS, component CSS, and flag images use `asset!("/assets/...")`; the shared [`App`](../packages/ui/src/client/app.rs) links header/footer styles before the routed shell renders to reduce first-paint flicker. | [Assets](https://dioxuslabs.com/learn/0.7/essentials/ui/assets/) |
| 8 | Async resources | ✅ | [`Page01`](../packages/ui/src/client/pages/page01.rs) loads template data with `use_resource` and reruns from a refresh signal. | [Data fetching](https://dioxuslabs.com/learn/0.7/essentials/basics/resources/) |
| 9 | Effects and timers | ✅ | [`Page01`](../packages/ui/src/client/pages/page01.rs) updates cache/toast state with `use_effect`, and [`toast`](../packages/ui/src/client/components/toast.rs) owns platform-specific timeout cleanup. | [Effects](https://dioxuslabs.com/learn/0.7/essentials/basics/effects/) |
| 10 | Browser local storage | ✅ | [`storage_service`](../packages/ui/src/client/services/storage_service.rs) stores browser theme, language, and template data snapshots in localStorage. | [Web platform](https://dioxuslabs.com/learn/0.7/guides/platforms/web/) |
| 11 | Native SQLite cache | ✅ | [`database_service`](../packages/ui/src/client/services/database_service.rs) reads native SQLite data under `data/` for non-wasm builds. | [Async](https://dioxuslabs.com/learn/0.7/essentials/basics/async/) |
| 12 | First-time DB creation | ✅ | [`create_database_if_missing()`](../packages/ui/src/client/services/database_service.rs) is the boilerplate location for first-time native schema creation and seed-row insertion. Normal reads do not recreate or reseed an existing DB. | [Async](https://dioxuslabs.com/learn/0.7/essentials/basics/async/) |
| 13 | Localization | ✅ | [`localization_service`](../packages/ui/src/client/services/localization_service.rs) loads Fluent bundles for English, Spanish, Portuguese, and French. | [Internationalization crate](https://crates.io/crates/dioxus-i18n) |
| 14 | Theme persistence | ✅ | [`storage_service`](../packages/ui/src/client/services/storage_service.rs) persists the two theme choices for web and desktop. | [Signals](https://dioxuslabs.com/learn/0.7/essentials/basics/signals/) |
| 15 | Web and desktop targets | ✅ | [`packages/web`](../packages/web/src/main.rs) and [`packages/desktop`](../packages/desktop/src/main.rs) launch the same shared UI; the root [`index.html`](../index.html) and desktop WebView config use the default dark shell color before the app renders. | [Platforms](https://dioxuslabs.com/learn/0.7/guides/platforms/) |
| 16 | Server contact probe | ✅ | [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs) includes a Server button that probes the same-origin web server and reports success or failure through the shared toast. | [Web platform](https://dioxuslabs.com/learn/0.7/guides/platforms/web/) |
| x1 | Forms | ❌ | No in-app form flow is implemented yet. Add one when a future template spec needs form handling. | [Forms](https://dioxuslabs.com/learn/0.7/essentials/fullstack/forms/) |
| x2 | Server functions | ❌ | The template currently has no server-function probe or fullstack server path. Add one only when a spec requires server execution. | [Server functions](https://dioxuslabs.com/learn/0.7/essentials/fullstack/server_functions/) |
| x3 | Dynamic routes | ❌ | The template currently has only static routes. Add dynamic route fields to `Route` when project content needs parameterized pages. | [Dynamic routes](https://dioxuslabs.com/learn/0.7/essentials/router/routes/) |
